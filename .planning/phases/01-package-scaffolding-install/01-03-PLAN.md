**references/backtest_engine.py** -- Structural skeleton per decisions D-15, D-16. This is the pattern Claude fills in per strategy.

```python
"""
Backtest Engine Skeleton -- Claude fills in calculate_signal() only.
DO NOT modify: event loop, position management, or metrics integration.

Usage:
    from backtest_engine import run_backtest
    results = run_backtest(df, params)
"""
import pandas as pd
import numpy as np
from pathlib import Path
import sys
import json

# Import fixed metrics module
sys.path.insert(0, str(Path.home() / '.pmf' / 'references'))
from metrics import compute_all_metrics
```

The file must contain:
1. `calculate_signal(history: pd.DataFrame, params: dict) -> str` -- stub that returns 'hold'. Claude replaces this per strategy. Returns 'long', 'short', 'close', or 'hold'.
2. `run_backtest(df: pd.DataFrame, params: dict) -> dict` -- the event loop:
   - Initializes: `equity = [params.get('initial_capital', 10000)]`, `trades = []`, `position = None`
   - Loops `for i in range(1, len(df)):`
     - `history = df.iloc[:i+1]` -- ONLY past + current bar (anti-lookahead)
     - `signal = calculate_signal(history, params)`
     - `execution_price = df.iloc[i]['open']` -- next-bar execution (current bar's open simulates previous bar's signal executing at next open)
     - Position management: open on long/short signal, close on close signal or opposite signal
     - Commission deduction: `params.get('commission', 0.001)` as fraction of trade value
     - Track trades: each trade dict has entry_time, exit_time, entry_price, exit_price, size, side, pnl, commission
     - Track equity: append current equity after each bar
   - After loop: `return compute_all_metrics(trades=trades, equity_curve=np.array(equity), trading_days=params.get('trading_days', 252))`
3. `save_iteration_artifacts(results: dict, params: dict, iteration: int, output_dir: str)` -- saves params.json, metrics.json to output directory

Comments in the code must explicitly state the anti-lookahead rules:
- "RULE 1: history[:i+1] -- signal sees only past and current bar"
- "RULE 2: execution at df.iloc[i]['open'] -- next bar's open price"
- "RULE 3: No future data in calculate_signal()"

Must contain three functions:
1. `load_ccxt(exchange: str, symbol: str, timeframe: str, since: str, limit: int = 1000) -> pd.DataFrame`
   - Uses ccxt library to fetch OHLCV data
   - Returns DataFrame with columns: timestamp, open, high, low, close, volume (all lowercase)
   - Sets timestamp as DatetimeIndex
   - Validates: no NaN values in OHLCV, no duplicate timestamps, sorted ascending
   - Handles pagination for large requests (ccxt returns max ~1000 candles per call)
   - Error handling: clear message if exchange not found, symbol not found, rate limited

2. `load_yfinance(ticker: str, start: str, end: str, interval: str = '1d') -> pd.DataFrame`
   - Uses yfinance library
   - Returns same DataFrame format as load_ccxt (lowercase columns: open, high, low, close, volume)
   - Renames yfinance's capitalized columns to lowercase
   - Validates: no NaN rows (drops them with warning), sorted ascending
   - Error handling: clear message if ticker not found or date range invalid

3. `load_csv(filepath: str, date_column: str = 'date', date_format: str = None) -> pd.DataFrame`
   - Reads CSV file, parses date column as DatetimeIndex
   - Normalizes column names to lowercase
   - Validates: required columns (open, high, low, close) present, no NaN in OHLCV
   - Error handling: clear message if file not found, required columns missing

Each function includes a `validate_ohlcv(df: pd.DataFrame) -> pd.DataFrame` helper call that:
- Checks for NaN values (logs warning, drops rows)
- Checks for duplicate timestamps (drops duplicates)
- Ensures ascending sort by timestamp
- Verifies open/high/low/close are numeric
