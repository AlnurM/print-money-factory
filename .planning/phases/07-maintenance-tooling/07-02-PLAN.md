The exact preamble content to insert in each workflow (per D-05 through D-09):

Section heading: `## Preamble: Version Check`

Body text: `**This check is silent and best-effort. It MUST NOT block or delay the workflow.**`

Step 1: Check if `~/.pmf/.last_version_check` exists and its modification time is within the last 24 hours.
  Run via Bash: `find ~/.pmf/.last_version_check -mtime -1 2>/dev/null`
  If the command outputs the filename, the check was done recently -- SKIP the rest of this preamble, proceed to the next section.
  If the command outputs nothing (file missing or older than 24h), continue.

Step 2: Read `~/.pmf/.version` to get the current installed version.
  Parse the JSON and extract the `version` field.
  If the file does not exist, SKIP the rest of this preamble silently.

Step 3: Check npm for the latest version.
  Run via Bash: `npm view print-money-factory version 2>/dev/null`
  If the command fails (network error, package not found), SKIP silently -- no error, no notice (per D-08).

Step 4: Compare versions.
  If the npm version differs from the installed version, display exactly:
  `Update available: v{current} -> v{latest}. Run /brrr:update`
  If versions match, display nothing.

Step 5: Update the timestamp file.
  Run via Bash: `touch ~/.pmf/.last_version_check`
  This gates the check to once per 24 hours.

**Important implementation details:**
- The preamble must appear as the FIRST section after the header in every workflow
- For `execute.md` which has "Preamble: Sequence Validation", the version check becomes the first preamble and the sequence validation becomes the second
- Do NOT change any existing workflow content below the inserted preamble -- only add the new section
- A horizontal rule before the preamble heading is required (matches workflow section separator pattern)
- Use `2>/dev/null` on all Bash commands to suppress errors (per D-08)
