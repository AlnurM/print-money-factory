**Step 2: Define Parameter Space**
- Per decision D-15: Build the complete parameter space from discuss output

  1. Confirm the range (min, max) and step size from discuss output
  2. If discuss did not specify step size, suggest reasonable defaults:
     - Integer params: step = 1 (or round to nearest meaningful increment)
     - Float params: step = (max - min) / 20 (approximately 20 steps)
     - Period params (e.g., SMA length): step = 1 for small ranges, 5 for large ranges (>50)
  3. Calculate total combinations for each parameter

  - Record all constraints in the plan

**Step 3: Parameter Budget Check**
- Per decision D-13 (PLAN-07): Calculate parameter budget
- Compute:
  - Total free parameters: count of params to optimize
  - Total combinations: product of (range/step) for each param
  - Estimated data points: (date range in bars) based on timeframe
  - Minimum trades from success criteria (default 30)
  - Ratio: data_points / (total_combinations * free_params)
- Warning thresholds:
  - If free_params > 5: WARN "You have {N} free parameters. More than 5 significantly increases overfitting risk. Consider fixing some parameters."
  - If total_combinations > 100 * min_trades: WARN "You're testing {N} combinations against a {min_trades}-trade minimum. Statistical significance is questionable."
  - If total_combinations > 10000: WARN "Grid search with {N} combinations will be slow. Random search or walk-forward recommended."
- Per D-13: Recommend reducing, but ALLOW user override
- Display the budget analysis clearly:
  ```
  --- Parameter Budget ---
  Free parameters: {N}
  Total combinations: {N}
  Estimated data bars: ~{N}
  Min trades required: {N}

  {WARNING if applicable}
  {RECOMMENDATION if applicable}

  Continue with this parameter space? (yes / modify)
  ```
