**HEADER:**
```markdown
# Workflow: new-milestone

Create a new trading strategy milestone through guided conversation.
```

**SECTION 1: Preamble — Active Milestone Check (MILE-04, D-12)**
Instructions for Claude to:
1. Check if `.pmf/STATE.md` exists in the current working directory (use Read tool)
2. If exists, check the Status field
3. If status is "IN PROGRESS", STOP and show error:

```
[STOP] Cannot create a new milestone -- you already have an active one.

Current milestone: {name from STATE.md}
Status: IN PROGRESS

To start a new milestone, first:
- Complete the current one: /brrr:verify --approved
- Or abandon it: manually set Status to "ABANDONED" in .pmf/STATE.md
```

4. If status is not "IN PROGRESS" (APPROVED, ABANDONED, or file doesn't exist), proceed

**SECTION 2: Preamble — Context File Scan (CTXT-01, CTXT-02, CTXT-03, D-02, D-15, D-16, D-17, D-18)**
Instructions for Claude to:
1. Check if `.pmf/context/` directory exists (use Glob or Bash)
2. If `--no-context` argument was passed, skip this section
3. If directory exists, list all files in it
4. Compare against Processed Context table in STATE.md (if it exists) to find NEW files
5. For each new file (up to 5 per invocation):
   - Images (PNG, JPG, JPEG, GIF, WEBP): Use Read tool to view the image. Describe in detail what you see (chart patterns, indicators, annotations, timeframe, asset). State your interpretation of how this relates to a trading strategy. Ask user: "Is this understanding correct? Anything to add or correct?" Wait for response.
   - PDF files: Use Read tool with pages parameter. Summarize content relevant to trading. Ask for confirmation.
   - Text files (.txt, .md, .csv): Read content directly. Summarize what's relevant. Ask for confirmation.
6. After each confirmation, note the file, date, and description for recording in STATE.md later
7. If more than 5 new files found, process first 5 and note: "X more files detected -- run a command again to process the rest."

**SECTION 3: Setup — Create Directory Structure (D-06)**
Instructions for Claude to:
1. Create `.pmf/` directory if it doesn't exist (use Bash: `mkdir -p .pmf/context .pmf/phases`)
2. This ensures context/ and phases/ subdirectories exist too

**SECTION 4: Step 1 — Strategy Idea (MILE-01, D-01)**
Title: "## Step 1: Collect Strategy Idea"
Instructions for Claude to:
1. Ask the user to describe their trading idea
2. Listen for and identify: strategy type (trend-following, mean-reversion, breakout, custom), asset class (crypto, stocks, forex), specific indicators or signals, timeframe hints
3. If the idea is vague, ask follow-up questions. If it mixes multiple concepts, help clarify: "You mentioned X, Y, and Z -- these are different approaches. Which is the core idea, and which are filters?"
4. If context files were processed in Section 2, reference what was learned from them
5. Do NOT proceed until you have a clear, single-thesis hypothesis
6. Record the detected strategy type for use in Step 5

**SECTION 5: Step 2 — Scope Selection (MILE-02, D-03)**
Title: "## Step 2: Scope Selection"
Instructions for Claude to:
1. Based on strategy complexity, suggest a default scope:
   - Simple strategies (single indicator, classic pattern): Strategy logic, Backtesting & optimization, PineScript export (3 items)
   - Complex strategies (multiple signals, custom logic, multi-timeframe): Strategy logic, Backtesting & optimization, Parameter tuning, Risk management, PineScript export (5 items)
2. Present the full list of 6 items with your suggestion pre-checked:
   - Strategy logic (entry/exit rules)
   - Backtesting & optimization
   - Parameter tuning
   - Risk management
   - PineScript export
   - MD instructions export
3. Let user modify the selection

**SECTION 6: Step 3 — Scope Splitting Check (MILE-03, D-05)**
Title: "## Step 3: Scope Splitting Check"
Instructions for Claude to:
1. If user selected 5 or more scope items, suggest splitting into phases
2. Recommend what goes in v1 (core: strategy + backtest + PineScript) vs v2 (tuning, risk, MD export)
3. But respect user override -- if they say something like "I know it's big, do it anyway", proceed with full scope. Splitting is a suggestion only, NOT enforced.

**SECTION 7: Step 4 — Data Source**
Title: "## Step 4: Data Source"
Instructions for Claude to:
1. Based on the asset class identified in Step 1, ask about specific asset and determine data source:
   - Crypto: Which exchange? Which pair? -> ccxt
   - Stocks: Which ticker? Daily or intraday? -> yfinance (daily) or polygon (intraday, requires API key)
   - Forex: Which pair? Daily or intraday? -> yfinance (daily) or dukascopy (intraday)
2. Ask about timeframe (1m, 5m, 15m, 1H, 4H, 1D, 1W)
3. Ask about date range (suggest reasonable defaults: 3-4 years for daily, 1-2 years for intraday)

**SECTION 8: Step 5 — Success Criteria (D-04)**
Title: "## Step 5: Success Criteria"
Instructions for Claude to:
1. Based on strategy type detected in Step 1, offer defaults:
   - Trend-following: Sharpe > 1.2, Max DD < 30%, Min trades 30
   - Mean-reversion: Sharpe > 1.5, Max DD < 20%, Min trades 30
   - Breakout: Sharpe > 1.0, Max DD < 35%, Min trades 30
   - Custom: Sharpe > 1.0, Max DD < 30%, Min trades 30
2. Present defaults and let user adjust any values
3. Record: primary metric (Sharpe target), max drawdown target, minimum trade count, strategy type

**SECTION 9: Step 6 — Summary and Confirmation**
Title: "## Step 6: Summary and Confirmation"
Instructions for Claude to:
1. Display a formatted summary of ALL collected information:
   - Strategy name/hypothesis
   - Strategy type
   - Asset, exchange, timeframe, date range
   - Scope items selected
   - Success criteria
   - Context files processed (if any)
2. Ask user to confirm or request changes
3. If changes requested, go back to the relevant step

**SECTION 10: Step 7 — Create Files (D-07, D-08, STAT-01, STAT-04, STAT-05)**
Title: "## Step 7: Create Files"
Instructions for Claude to:
1. Read `~/.pmf/templates/STRATEGY.md` (the template)
2. Fill all template variables with collected information
3. Write to `.pmf/STRATEGY.md`
4. Read `~/.pmf/templates/STATE.md` (the template)
5. Fill template variables:
   - Milestone name from the hypothesis
   - Status: IN PROGRESS
   - Current Phase: 1
   - Current Step: discuss
   - Created/Last Updated: current date
   - Scope: checked items from Step 2
   - Data Source: from Step 4
   - Success Criteria: from Step 5
   - Phases: Phase 1 with all steps unchecked (discuss, research (optional), plan, execute, verify)
   - Determine phase list based on scope -- if research is relevant for this strategy type, include it; otherwise mark as optional
   - Best Results: empty table
   - Processed Context: entries from Section 2 (if any), or empty table
   - History: single entry "### {date} -- Milestone created" with hypothesis and scope summary
6. Write to `.pmf/STATE.md`
7. Display confirmation: "Milestone created! Next step: /brrr:discuss"
