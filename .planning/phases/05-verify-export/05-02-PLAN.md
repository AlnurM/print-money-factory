The file should contain these sections:

**1. Header**
```
# PineScript v5 Syntax Rules
> Reference for Claude when generating PineScript output. Read this before writing any .pine files.
> Also read: templates/pinescript-template.pine and references/pinescript-examples/*.pine
```

**2. Version Declaration**
- Always start with `//@version=5`
- PineScript v5, NOT v6. Add comment: `// Note: This is PineScript v5. For v6 migration, see: https://www.tradingview.com/pine-script-docs/migration-guides/to-pine-version-6/`

**3. Strategy vs Indicator**
- Per D-17: Generate TWO versions of every strategy
- Strategy version: `strategy("Name", overlay=true, initial_capital=N, ...)` -- uses `strategy.entry()`, `strategy.close()`, `strategy.exit()`
- Indicator version: `indicator("Name Signals", overlay=true)` -- uses `plotshape()`, `alert()`, `alertcondition()`
- CRITICAL: `strategy.*` functions are ONLY available in strategy() scripts. NEVER call `strategy.entry()` in an indicator() script. This is the #1 PineScript generation error.
- Per D-19: Output files are `pinescript_v5_strategy.pine` and `pinescript_v5_indicator.pine`

**4. Input Functions**
- `input.int(defval, title, minval, maxval)` for integer params
- `input.float(defval, title, minval, maxval, step)` for float params
- `input.bool(defval, title)` for boolean flags
- `input.string(defval, title, options)` for string selections
- All strategy parameters should be inputs so users can adjust in TradingView UI

**5. Common Indicator Functions (ta namespace)**
- `ta.ema(source, length)` -- Exponential Moving Average
- `ta.sma(source, length)` -- Simple Moving Average
- `ta.rsi(source, length)` -- RSI
- `ta.atr(length)` -- Average True Range
- `ta.crossover(a, b)` -- returns true when a crosses above b
- `ta.crossunder(a, b)` -- returns true when a crosses below b
- `ta.highest(source, length)` -- highest value over N bars
- `ta.lowest(source, length)` -- lowest value over N bars
- `ta.stoch(close, high, low, length)` -- Stochastic
- `ta.macd(source, fastLength, slowLength, signalLength)` -- returns [macdLine, signalLine, histogram]

**6. Strategy Execution Functions**
- `strategy.entry(id, direction)` -- direction is `strategy.long` or `strategy.short`
- `strategy.close(id)` -- close position by entry ID
- `strategy.exit(id, from_entry, stop, limit)` -- set stop loss and/or take profit
- Always use `if` blocks for conditional entries (not `when` parameter which is deprecated in v6):
```pine
if longCondition
    strategy.entry("Long", strategy.long)
```

**7. Indicator Signal Functions**
- `plotshape(condition, title, location, color, style, size)` -- visual signal marker
  - location: `location.belowbar`, `location.abovebar`, `location.absolute`
  - style: `shape.triangleup`, `shape.triangledown`, `shape.circle`, `shape.cross`
  - size: `size.small`, `size.normal`, `size.large`
- `alert(message, freq)` -- fires alert during script execution
  - freq: `alert.freq_once_per_bar`, `alert.freq_once_per_bar_close`, `alert.freq_all`
- `alertcondition(condition, title, message)` -- creates user-configurable alert (legacy but widely used)
- Include BOTH `alert()` and `alertcondition()` for maximum compatibility

**8. Plotting Functions**
- `plot(series, title, color, linewidth)` -- plot a line
- `bgcolor(color)` -- background color
- `barcolor(color)` -- bar color
- Colors: `color.red`, `color.green`, `color.blue`, `color.orange`, `color.new(color, transparency)` (0=opaque, 100=transparent)

**9. Common Patterns**
- Trailing stop: track highest since entry, set stop at highest - N*ATR
- Breakout: `ta.highest(high, length)` for N-bar high breakout
- Mean reversion: RSI oversold/overbought with Bollinger Band confirmation
- Trend following: MA crossover with ADX filter

**10. v5 vs v6 Migration Notes**
- v6 removes `when` parameter from strategy functions
- v6 changes: `int`/`float` no longer implicitly cast to `bool` (use explicit comparisons)
- v6 removes 9000-trade limit
- v5 is stable and broadly supported; generate v5 code
