Create `workflows/doctor.md` -- a behavioral markdown workflow that Claude executes step by step. This is the core of MANT-01.

Structure the workflow with these sections (per D-01 through D-04):

**Header:**
Title: "# Workflow: doctor"
Description: "Run diagnostic checks on the Print Money Factory installation. This workflow is READ-ONLY -- it does not modify any files."

**Section 1: Check Python Version (per D-02)**
1. Run `python3 --version` via Bash
2. Parse the version number. If >= 3.10, report `[PASS] Python {version} detected`
3. If < 3.10 or not found, report `[FAIL] Python >= 3.10 required -- install from python.org` (per D-03)

**Section 2: Check venv (per D-02)**
1. Check if `~/.pmf/venv/bin/python` exists (use `ls` via Bash)
2. If exists: `[PASS] venv exists at ~/.pmf/venv/`
3. If missing: `[FAIL] venv missing -- run npx print-money-factory install` (per D-03)

**Section 3: Check core imports (per D-02)**
This is the critical check -- must use actual imports, not file existence.

For each library in the list: pandas, numpy, ccxt, yfinance, plotly, optuna, ta, matplotlib, scipy, jinja2:
Run via Bash: `~/.pmf/venv/bin/python -c "import {library}"` for each one.
If succeeds: `[PASS] import {library}`
If fails: `[FAIL] import {library} -- run ~/.pmf/venv/bin/pip install -r requirements.txt` (per D-03)

Provide a single Bash command that tests all imports at once using a Python script that tries each import in a try/except and prints [PASS]/[FAIL] per library.

If venv check in Section 2 failed, skip this section and report all 10 imports as [FAIL] with the venv fix suggestion.

**Section 4: Check command files (per D-02)**
1. Use Glob to list all files in `~/.claude/commands/brrr/`
2. Check that the expected commands exist: discuss.md, doctor.md, execute.md, new-milestone.md, plan.md, research.md, status.md, update.md, verify.md
3. For each found: no output needed (pass silently)
4. For any missing: `[FAIL] Missing command: {name}.md -- run npx print-money-factory install`
5. If all present: `[PASS] Command files (9 files in ~/.claude/commands/brrr/)`

**Section 5: Check workflow files (per D-02)**
1. Use Glob to list all files in `~/.pmf/workflows/`
2. Check that expected workflows exist: discuss.md, doctor.md, execute.md, new-milestone.md, plan.md, research.md, status.md, verify.md
3. If all present: `[PASS] Workflow files (8 files in ~/.pmf/workflows/)`
4. For any missing: `[FAIL] Missing workflow: {name}.md -- run npx print-money-factory install`

**Section 6: Check reference files (per D-02)**
1. Check if `~/.pmf/references/` directory exists and is non-empty
2. If exists and has files: `[PASS] Reference files present in ~/.pmf/references/`
3. If missing or empty: `[FAIL] Reference files missing -- run npx print-money-factory install`

**Section 7: Summary (per D-04)**
1. Count total checks run and total passes
2. Display: `{X}/{Y} checks passed`
3. If all passed: `Verdict: HEALTHY`
4. If any failed: `Verdict: NEEDS ATTENTION`
5. If any failed, add: "Run the suggested fix commands above to resolve issues."

**Important constraints:**
- This workflow is READ-ONLY -- it does not modify any files
- Use the exact `[PASS]` and `[FAIL]` prefix format (per D-01)
- Each FAIL line includes a fix suggestion (per D-03)
- The workflow must handle the case where venv does not exist gracefully (skip import checks, report venv FAIL)
