<decisions>
## Implementation Decisions

### Guide content & structure
- **D-01:** The guide has these sections (in order):
  1. **Prerequisites** — what you need before starting (exchange account, API keys, Python environment)
  2. **Platform Setup** — step-by-step for the detected platform (exchange API, broker API, or MT5/OANDA)
  3. **Strategy Configuration** — how to configure the approved parameters in the live system
  4. **Order Types & Execution** — which order types to use (limit vs market), slippage handling
  5. **Risk Management** — position sizing rules, max exposure, drawdown circuit breaker
  6. **Monitoring & Alerts** — what to watch, how to set up alerts, when to intervene
  7. **Go-Live Checklist** — final pre-launch verification steps (distinct from the existing live-checklist.md which is generic)
- **D-02:** Each section includes concrete examples with the actual strategy parameters (not generic placeholders). The guide references the approved params from `phase_N_best_result.json`.
- **D-03:** Code snippets are at the "pattern" level — showing API call structure, not a full runnable bot. Per Out of Scope: "Bot-building guide stays at code pattern level, not full bot."

### Platform detection
- **D-04:** Platform is detected from the asset class in STRATEGY.md:
  - Crypto assets (BTC, ETH, etc.) → exchange API guide (ccxt patterns, Binance/Bybit examples)
  - Stock assets (SPY, AAPL, etc.) → broker API guide (Alpaca, Interactive Brokers patterns)
  - Forex assets (EUR/USD, GBP/JPY, etc.) → MT5/OANDA guide
- **D-05:** The AI reads STRATEGY.md to detect asset class automatically. No user prompt needed.
- **D-06:** If asset class is ambiguous, the guide includes a brief section for each applicable platform rather than asking.

### Tone & audience
- **D-07:** Target audience: experienced trader who knows their platform but hasn't automated this specific strategy before. Same practitioner tone as trading-rules.md (D-10 from Phase 5).
- **D-08:** Use direct, second-person language ("Set your API key...", "Configure position size to..."). No hedging or disclaimers beyond the standard risk warning at the top.
- **D-09:** Include a standard risk warning at the very top: "This guide assumes you've validated the strategy in backtesting. Past performance does not guarantee future results. Start with paper trading or minimal position sizes."

### Claude's Discretion
- Exact code snippet examples for each platform
- Whether to reference the existing PineScript exports for TradingView-based automation
- Level of detail in the monitoring section
- Whether to include a "Common Pitfalls" subsection

</decisions>
