**HEADER:**
```markdown
# Workflow: status

Display milestone progress as an ASCII tree with next step recommendation.
This workflow is READ-ONLY -- it does not modify any files.
```

**SECTION 1: Check for Active Milestone**
Instructions for Claude to:
1. Check if `.pmf/STATE.md` exists in the current working directory (use Read tool)
2. If not found, display: "No active milestone. Run /brrr:new-milestone to get started." and STOP.
3. If found, read the entire file

**SECTION 2: Context Scan (optional)**
Instructions for Claude to:
1. If `--no-context` flag is present, skip
2. Otherwise, check `.pmf/context/` for new files not in the Processed Context table
3. For each new file (up to 5):
   - Read and describe the file (images via multimodal, text directly, PDFs via pages)
   - Ask user to confirm understanding
   - Note: Since status has no Write permission, inform user: "New context file detected. Context will be processed on your next /brrr: command that modifies state."
4. This is informational only -- status does not update STATE.md

**SECTION 3: Parse STATE.md**
Instructions for Claude to extract:
1. From ## Status: milestone name, status, current phase number, current step
2. From ## Data Source: asset, timeframe
3. From ## Success Criteria: primary target (Sharpe), max drawdown target, strategy type
4. From ## Phases: for each phase, for each step -- completion status (checkbox state), inline metrics/dates if present
5. From ## Best Results: table rows with phase metrics

**SECTION 4: Render ASCII Tree (D-19, D-22, MILE-05)**
Instructions for Claude to render the tree in this exact format:

```
Print Money Factory -- Status

  Milestone: {name} | {asset} {timeframe}
  Target: Sharpe > {sharpe_target} | Max DD < {maxdd_target}%
  -------------------------------------------
  Phase 1
    [DONE] discuss    {description or date}
    [SKIP] research   skipped
    [DONE] plan       {description}
    [DONE] execute    Sharpe {value} | MaxDD {value} | {N} trades
    [DONE] verify     -> {verdict}
  Phase 2
    [DONE] discuss    {description}
    [WIP]  plan       in progress...
    [    ] execute
    [    ] verify

  Next step: /brrr:{command}
```

Icon rules:
- Completed step `[x]` in STATE.md -> `[DONE]`
- Current step (matches Current Step field) -> `[WIP]`
- Skipped step (marked "skipped" or "optional" and bypassed) -> `[SKIP]`
- Not started -> `[    ]` (4 spaces)

For the execute step specifically: if completed, show inline metrics from the checkbox line or Best Results table: "Sharpe {value} | MaxDD {value} | {N} trades"
For the verify step specifically: if completed, show the verdict: "-> {verdict}" (mint, debug, etc.)

Align step names and descriptions in columns for readability.

**SECTION 5: Determine Next Step (D-20)**
Instructions for Claude to:
1. Read Current Phase and Current Step from STATE.md
2. The next step IS the current step (it's what needs to be done next)
3. Display at the bottom: "Next step: /brrr:{current_step}"
4. If all phases are complete (status is APPROVED), display: "Milestone complete! Strategy approved and exported."
5. If status is ABANDONED, display: "Milestone abandoned. Run /brrr:new-milestone to start fresh."

**SECTION 6: Best Results Summary (STAT-02)**
Instructions for Claude to:
If the Best Results table has any rows, display them after the tree:

```
  Best Results:
  | Phase | Sharpe | Max DD | Trades | Verdict |
  |-------|--------|--------|--------|---------|
  | 1     | 1.58   | -17.2% | 52     | debug   |
```

If no results yet, skip this section.

**IMPORTANT CONSTRAINT:** This workflow is READ-ONLY. The status command's allowed-tools list does NOT include Write. Do NOT include any instructions to modify STATE.md or any other file. If context files are found, note them but do not process them into state.
