Create references/report_generator.py -- a Python module that the verify workflow will invoke to generate the HTML report. This is a REFERENCE file (like metrics.py, backtest_engine.py) that gets copied to ~/.pmf/ during install.

The script must:

**1. Main entry point: `generate_report(phase_dir, strategy_name, targets, output_path, template_path, cache_dir=None, plan_path=None)`**

Parameters:
- `phase_dir`: Path to `.pmf/phases/` (contains phase_N_execute/ subdirs and phase_N_best_result.json)
- `strategy_name`: String for report title
- `targets`: Dict of target metrics (e.g., {"sharpe_ratio": 1.5, "max_drawdown": -0.15})
- `output_path`: Where to write the HTML file
- `template_path`: Path to report-template.html
- `cache_dir`: Path to `.pmf/cache/` for OHLCV data (optional, regime/benchmark skipped if None)
- `plan_path`: Path to phase_N_plan.md (optional, to detect grid search for heatmap)

**2. Data loading functions:**

`load_iteration_artifacts(phase_dir, phase_num)`:
- Glob for `phase_{phase_num}_execute/iter_*_verdict.json`
- For each iteration, load verdict JSON (has params, metrics, hypothesis, verdict)
- Also load `iter_*_metrics.json` and `iter_*_params.json` as fallback
- Look for `iter_*_oos_metrics.json` for OOS Sharpe
- Return list of dicts sorted by iteration number

`load_best_result(phase_dir, phase_num)`:
- Load `phase_{phase_num}_best_result.json`
- Return dict with best_params, is_metrics, oos_metrics, stop_reason

**3. Analytics computation:**

`compute_equity_curve(best_result, ohlcv_df)`:
- Re-run backtest with best params to get bar-level equity curve (per Research open question 1 -- re-running is safer than modifying metrics.py)
- Import backtest_engine and metrics from the same references directory
- Return dict with dates (list) and equity values (list)
- Also compute buy & hold equity: initial_capital * (close / close[0])

`compute_drawdown_series(equity_curve)`:
- From equity array, compute running max and drawdown percentage at each bar
- Return dict with dates and drawdown values (negative percentages)
- Also return max_drawdown_pct (single float, most negative value)

`classify_regimes(ohlcv_df, sma_period=50, adx_period=14, adx_threshold=25)`:
- Per D-03: SMA slope + ADX classification
- Use `ta.trend.ADXIndicator` (already in venv)
- Bull: SMA slope > 0 AND ADX > threshold
- Bear: SMA slope < 0 AND ADX > threshold
- Sideways: ADX <= threshold
- Fill NaN with 'sideways' (warmup bars, per Research pitfall 4)
- Return pd.Series with values 'bull', 'bear', 'sideways'

`compute_regime_stats(trades, regimes_series, ohlcv_df)`:
- For each trade, determine which regime it occurred in (use entry date to look up regime)
- Compute per-regime: bar count, trade count, win rate, avg P&L, total P&L, contribution %
- Return list of dicts with name, bars, trades, win_rate, avg_pnl, total_pnl, contribution, color
- Colors: bull=#16a34a, bear=#dc2626, sideways=#f59e0b

`compute_benchmark_stats(strategy_equity, buyhold_equity)`:
- Compute daily returns for both
- Use `scipy.stats.linregress(benchmark_returns, strategy_returns)`
- alpha = intercept * 252 (annualize), beta = slope, r_squared = r_value**2
- Return dict: alpha, beta, r_squared

**4. Template rendering:**

`render_report(template_path, output_path, **data)`:
- Read template file
- Use `jinja2.Template(template_text).render(**data)`
- Write rendered HTML to output_path
- Return output_path

**5. Data formatting helpers:**

`format_metrics_cards(metrics, targets)`:
- Take metrics dict and targets dict
- For each metric, create card dict: label (UPPERCASE), value (formatted per UI-SPEC), color (#16a34a if meets target, #dc2626 if below, #333333 if no target)
- Metric formats per UI-SPEC copywriting contract:
  - sharpe_ratio: "1.45" (2dp)
  - max_drawdown: "-12.3%" (1dp, always negative)
  - win_rate: "58.2%" (1dp)
  - profit_factor: "1.82" (2dp)
  - total_trades: "147" (integer)
  - net_pnl: "+$12,450" or "-$3,200" (dollar sign, comma, sign prefix)
  - sortino_ratio: "2.14" (2dp)
  - calmar_ratio: "1.67" (2dp)

`format_trades_table(trades)`:
- Sort chronologically by entry date
- For each trade: number, entry_date, exit_date, direction, entry_price, exit_price, pnl (formatted with $), pnl_pct (formatted with %), duration (days)
- Include pnl and pnl_pct as raw floats for CSS class conditionals

`format_iterations_table(iterations, best_iter_num)`:
- For each iteration: number, params (key params as compact string), sharpe_is, sharpe_oos, max_dd, trades count, verdict (MINT/PLATEAU/etc or "exploring")
- Mark best iteration with is_best=True for bold styling

`detect_grid_search(plan_path)`:
- Read plan_path, look for "grid" in optimization_method
- Return boolean

`build_heatmap_html(iterations, param_names)`:
- Only called if grid search detected
- Build 2D grid from iteration params/metrics
- Use plotly go.Heatmap with RdYlGn colorscale
- Return fig.to_html(full_html=False, include_plotlyjs=False)

**6. JSON serialization safety:**
- All numpy types must be converted to native Python types before Jinja2 rendering
- Helper: `def sanitize_for_json(obj)` that recursively converts numpy int64->int, float64->float, ndarray->list
