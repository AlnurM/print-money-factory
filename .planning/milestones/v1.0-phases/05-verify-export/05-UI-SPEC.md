# Phase 5 -- UI Design Contract

> Visual and interaction contract for the standalone HTML backtest report and terminal output. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (standalone HTML, not a frontend app) |
| Preset | not applicable |
| Component library | none -- plain HTML/CSS with plotly.js charts |
| Icon library | none -- text labels only |
| Font | `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif` (system font stack, already in template) |

**Rationale:** This phase produces a standalone HTML file opened in a browser and terminal output in Claude Code. No React, no build step, no component library. All styling is inline CSS in the Jinja2 template.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Metric card label-to-value gap, inline padding |
| sm | 8px | Metric card internal spacing, table cell padding (vertical), report header subtitle margin |
| md | 16px | Section internal padding between elements, metrics grid gap, section heading bottom margin |
| lg | 24px | Section padding (all sides), section bottom margin |
| xl | 32px | Report header vertical padding (30px rounded to 32px) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Table cell horizontal padding: 12px (between 8px and 16px, matching existing template `padding: 10px 12px`)
- Chart container min-height: 400px (functional constraint for plotly interactive area)
- Metric card padding: 16px (md token)

**Source:** Pre-populated from existing `templates/report-template.html` CSS values. Minor adjustments to align with 4px grid where existing values were off by 1-2px.

## Typography

**Source:** Pre-populated from existing template CSS. Only 2 weights used in practice: regular (400) for body content, semibold/bold (600/700) for emphasis.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fafafa` | Page background |
| Secondary (30%) | `#ffffff` | Section cards, metric card containers |
| Accent (10%) | `#2563eb` | Strategy equity line, primary data emphasis |
| Destructive | `#dc2626` | Negative P&L values, drawdown fill, losing trades, bear regime |
| Positive | `#16a34a` | Positive P&L values, winning trades, bull regime |

| Text secondary | `#666666` | Subtitle text, metric labels |

Accent reserved for: strategy equity curve line, metric values that meet targets, active iteration highlight in iteration table.

**Heatmap color scale (Claude's discretion per CONTEXT.md):**
- `RdYlGn` (Red-Yellow-Green) for parameter heatmap -- red for poor Sharpe, green for strong Sharpe. Standard financial convention.

**Regime colors:**
- Bull: `#16a34a` (positive green)
- Bear: `#dc2626` (destructive red)
- Sideways: `#f59e0b` (amber -- neutral regime, distinct from both positive and negative)

**Source:** Pre-populated from existing template CSS. Colors are locked in the template already. New additions (regime amber, muted gray) follow the same palette family.

|--------|-------|--------|

| Calmar Ratio | `CALMAR RATIO` | `1.67` (2 decimal) |

| Beta | `BETA` | `0.45` (2 decimal) |

| Condition | Visual Treatment |
|-----------|-----------------|

### Empty State

| Element | Copy |
|---------|------|
| No iterations found | `No iteration artifacts found in .pmf/phases/. Run /brrr:execute first.` |
| No cached data | `No cached OHLCV data found. Cannot generate regime breakdown or benchmark comparison.` |
| No grid search data | Parameter heatmap section is omitted entirely (no empty state -- section does not render) |
| Zero trades in best iteration | `Best iteration produced 0 trades. The strategy did not generate any signals in the test period. Consider debugging: entry conditions may be too restrictive.` |

### Error States

| Error | Copy |
|-------|------|
| Execute not complete | `Cannot verify: execute phase has not completed. Run /brrr:execute first.` |
| Report generation failed | `Report generation failed: {python_error}. Check that all iteration artifacts exist in .pmf/phases/.` |
| PineScript generation note | `Note: PineScript output has not been validated against TradingView. Paste into Pine Editor to verify before live use.` |
| Data file missing | `Cached data file not found at {path}. Regime breakdown and benchmark stats will be skipped.` |

### Destructive Actions

| Action | Confirmation Approach |
|--------|----------------------|
| --approved (close milestone) | AI presents full assessment first, then asks. No separate confirmation dialog -- the assessment IS the confirmation context. If targets not met, explicit warning before proceeding. |
| --debug (new phase cycle) | No confirmation needed -- debug is non-destructive (creates new cycle, preserves all existing artifacts). |

## Chart Interaction Contract

Since this phase's primary UI is interactive plotly charts, the following interaction contracts apply:

### Equity Curve Chart
| Property | Value |
|----------|-------|
| Chart type | Line chart (scatter, mode: lines) |
| Traces | 2: Strategy equity (`#2563eb`, width 2, solid) and Buy & Hold (`#9ca3af`, width 1, dashed) |
| X-axis | Date |
| Y-axis | Equity ($) |
| Hover | `x unified` -- shows both values at same date |
| Zoom | Built-in plotly box zoom + pan (default toolbar) |
| Margins | `{t: 10, r: 30, b: 40, l: 60}` |

### Drawdown Chart
| Property | Value |
|----------|-------|
| Chart type | Area chart (scatter, mode: lines, fill: tozeroy) |
| Line color | `#dc2626`, width 1 |
| Fill color | `rgba(220, 38, 38, 0.2)` |
| Max DD line | Horizontal line at max drawdown value, `#dc2626`, dashed |
| X-axis | Date |
| Y-axis | Drawdown (%) |
| Hover | `x unified` |
| Margins | `{t: 10, r: 30, b: 40, l: 60}` |

### Parameter Heatmap (conditional)
| Property | Value |
|----------|-------|
| Chart type | `go.Heatmap` |
| Color scale | `RdYlGn` (red = poor, green = strong) |
| Color bar label | `Sharpe Ratio` |
| Hover template | `{param1_name}: %{x}, {param2_name}: %{y}, Sharpe: %{z:.2f}` |
| Condition | Only rendered if optimization method was grid search |
| Margins | `{t: 10, r: 30, b: 40, l: 60}` |

### Trade List Table
| Property | Value |
|----------|-------|
| Sort order | Chronological (by entry date) |
| P&L coloring | Positive values: `.positive` class (`#16a34a`), negative: `.negative` class (`#dc2626`) |
| Columns | `#`, Entry Date, Exit Date, Direction, Entry Price, Exit Price, P&L ($), P&L (%), Duration |

### Iteration Table
| Property | Value |
|----------|-------|
| Sort order | By iteration number (ascending) |
| Columns | `#`, Parameters (key params), Sharpe (IS), Sharpe (OOS), Max DD, Trades, Verdict |
| Best row highlight | Bold text, no background change (keeps table scannable) |

### Regime Breakdown Table
| Property | Value |
|----------|-------|
| Columns | Regime, Bars, Trades, Win Rate, Avg P&L, Total P&L, Contribution % |
| Regime labels | `Bull` (green text), `Bear` (red text), `Sideways` (amber text) |

### Benchmark Stats Display
| Property | Value |
|----------|-------|
| Format | Metric cards (same style as top metrics grid) |
| Metrics shown | Alpha (annualized), Beta, R-squared |

## Layout Contract

### HTML Report Section Order (per D-01)

Decision-first layout -- answer "is it good?" before showing details:

1. **Report Header** -- Strategy name, subtitle
2. **Performance Metrics** -- Metric cards grid (the verdict at a glance)
3. **Equity Curve** -- Strategy vs buy & hold (how it grew)
4. **Drawdown** -- Risk visualization
5. **Optimization Iterations** -- How parameters evolved
6. **Parameter Sensitivity** -- Heatmap (conditional: grid search only)
7. **Trade Log** -- Individual trades with P&L
8. **Regime Breakdown** -- Bull/bear/sideways performance
9. **Benchmark Comparison** -- Alpha, beta, R-squared

### Responsive Behavior

| Breakpoint | Behavior |
|------------|----------|
| Desktop (default) | Metrics grid: `repeat(auto-fit, minmax(200px, 1fr))` -- 3-5 cards per row |
| Tablet (~768px) | Metrics grid auto-adjusts to 2-3 cards per row (handled by `auto-fit`) |
| Mobile (~480px) | Metrics grid stacks to 1-2 cards per row (handled by `auto-fit`) |
| Print | Cards: no box-shadow, 1px solid border. Charts: min-height 300px |

**Source:** Existing template already has responsive grid via `auto-fit` and print media query.
