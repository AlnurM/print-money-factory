**Preamble: Sequence Validation**
- Read `.pmf/STATE.md` to get current phase number N and verify discuss has not been completed for this phase

**Preamble: Mode Detection**
- Check if `--auto` flag was passed
- Check if this is a debug discuss (phase N > 1, meaning prior phase had execute/verify)
- Set mode: `first-discuss`, `debug-discuss`, or `auto`

**Step 1: Load Strategy Context**
- Read `.pmf/STRATEGY.md` -- extract hypothesis, strategy type, asset, timeframe, scope, success criteria, original idea
- Read `.pmf/STATE.md` -- extract current phase number
- If debug discuss: also read ALL prior phase artifacts:
  - `.pmf/phases/phase_M_discuss.md` (all prior M < N)
  - `.pmf/phases/phase_M_research.md` (if exists)
  - `.pmf/phases/phase_M_plan.md` (if exists)
  - `.pmf/phases/phase_N-1_best_result.json` (last execute results)
  - Read the verify report/diagnosis from the previous phase if it exists
- Quote the original hypothesis from STRATEGY.md to anchor the conversation

**Step 2: Strategy Discussion (first-discuss mode)**
- Per decision D-01: Follow-the-thread conversation style -- start open ("describe your strategy in more detail"), then follow what the user emphasizes
- Do NOT use a fixed topic checklist -- follow the user's energy
- As the conversation progresses, track which topics have been covered from the required set:
  1. Entry logic (what triggers a buy/sell)
  2. Exit logic (what triggers closing a position)
  3. Stop-loss rules (fixed %, trailing, ATR-based, none)
  4. Take-profit rules (fixed target, trailing, signal-based)
  5. Position sizing (fixed size, % of equity, Kelly criterion)
  6. Commission assumptions (per-trade flat or percentage, default by asset class: 0.1% crypto, 0.01% stocks, 0.005% forex)
  7. Parameter ranges (which params are fixed, which to optimize, ranges and step sizes)
- After the natural conversation winds down, check for gaps -- ask about any uncovered topics
- Per decision D-02: Ensure ALL 7 topics are captured before proceeding to output

**Step 2-alt: Strategy Discussion (auto mode)**
- Per decision D-03: detect strategy type from STRATEGY.md
- Apply type-specific defaults:
  - Trend-following: trailing stop at 2x ATR, no fixed TP, 2% risk per trade, 0.1% commission (crypto) or per asset class
  - Mean-reversion: fixed stop at 2%, TP at 1.5x stop distance, 1% risk per trade
  - Breakout: stop below breakout level, TP at 2x risk, 2% risk per trade
  - Custom: conservative defaults (2% stop, 2:1 RR target, 1% risk)
- Override with any specifics already in STRATEGY.md
- Present the full default set and ask: "These are the defaults for your {strategy_type} strategy. Any changes?" -- ONE question, then proceed
- If user says "looks good" or similar, proceed with defaults

**Step 2-debug: Strategy Discussion (debug-discuss mode)**
- Per decision D-04: Start with AI diagnosis, not from scratch
- Summarize what happened in the previous phase: what was tried, what the metrics were, what the verify diagnosis said
- Formulate a starting hypothesis: "Based on the previous results, I think the issue is X. Here's what I suggest changing: Y"
- Let the user react and discuss the proposed changes
- Per decisions D-17/D-18/D-19: Apply drift detection
  - Compare ALL proposed changes against STRATEGY.md Original Idea
  - Drift = new indicators/signals NOT in original STRATEGY.md OR fundamentally changed entry/exit logic, but ONLY if changes affect >50% of the strategy
  - Minor tweaks (filter adjustments, parameter range changes) are NOT drift
  - If drift detected: HARD GATE -- quote the original hypothesis from STRATEGY.md, show what changed, and force a choice:
    ```
    [DRIFT DETECTED] Your proposed changes significantly alter the original strategy.

    Original hypothesis: "{quoted from STRATEGY.md}"
    Proposed changes: {list changes that constitute drift}

    This looks like a different strategy. Choose:
    1. Stay within current scope (revert drift changes)
    2. Open a new milestone for this new approach (/brrr:new-milestone)

    Cannot continue until you choose.
    ```
  - Do NOT allow "continue anyway" -- the user must pick 1 or 2

**Step 3: Compile and Confirm Decisions**
- Display all collected decisions in a structured format:
  ```
  --- Strategy Decisions ---

  Entry: {logic}
  Exit: {logic}
  Stop-Loss: {type and value}
  Take-Profit: {type and value}
  Position Sizing: {method and value}
  Commissions: {type and value}

  Parameters to Optimize:
    {param_name}: {min} to {max}, step {step}
    ...

  Fixed Parameters:
    {param_name}: {value}
    ...
  ```
- Ask: "Does everything look correct? Say 'yes' to save, or tell me what to change."
- If changes requested, go back to the relevant topic

**Step 4: Write Output Artifact**

  ```markdown
  {For debug mode: what changed vs original, within scope / drift detected and resolved}

  ```
